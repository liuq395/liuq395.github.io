旋转-theta 角度就是旋转 theta 角的转置
对于旋转矩阵，逆矩阵就等于它的转置。
如果一个矩阵的逆矩阵等于转置矩阵，那么它是一个正交矩阵，所以旋转矩阵是正交矩阵

1. 三维变换

再次应用齐次坐标

  • 增加一个维度
    • 3D 点=(x, y, z,1)
    • 3D 向量=(x, y, z,0)
  • (x, y, z, w)(w!=0)表示的是 3D 的点(x/w, y/w, z/w)

使用 4*4 矩阵描述仿射变换

  • 平移的量仍然是最后一列tx,ty,tz,1
  • 在仿射变换的时候最后一行依然是 0 0 0 1
  • 顺序：对应到非齐次坐标是先线性变换（乘一个矩阵）再平移（加一个列向量）

旋转

  • 绕轴旋转：关于 y 轴和其他两轴写法上的一些不同
  • 一般性旋转：Rxyz(a,b,c) = Rx(a)Ry(b)Rz(c);
    想象一个飞机的三个方向旋转，飞机头可以上下，飞机可以左右转，飞机可以倾斜（想象机翼向一侧倾斜），这三个方向相当于绕 xyz 轴旋转，组合它们可以表示任意的旋转
  • 罗德里格斯旋转公式：给出一个旋转轴向量 n 和一个旋转角度 α，得到旋转矩阵
    • 旋转轴向量默认过原点
    • 如果想要任意也就是不一定过原点的轴怎么办？利用上节课的思想，先把所有东西都平移使得轴过原点，然后旋转，再全部平移回去，所得的结果是一样的
    • 两个向量的叉乘可以写成一个矩阵的形式乘一个另一个向量，最终也得到一个向量。就是公式中的 N 矩阵

2. 观测变换 Viewing transfomation

2.1 视图/相机变换 View/Camera Transformation

Intro : 三维空间转换为二维图片，怎么看

  • 什么是视图变换
  • 怎么拍一张照片：MVP 变换
    • Model transformation：找一个场景，安排入镜的人
    • ==View transformation：找一个好角度，调整相机的视角==
    • Projection transformation：“茄子！”
  • 如何进行视图变换
  • 定义相机
    • position 点 e
    • Look-at/gaze direction 向量 g
    • Up direction:向量 t，相机本身的旋转，有些像横屏竖屏（相机上一根草的指向）
  • 相对运动 - 如果相机和所有物体进行相同的移动，那么“照片”将会一直是相同的 - 让相机永远：放在原点，look at -Z, up at Y
    ==没听懂==
  • 怎么把相机移动到标准位置-数学表示变换矩阵 Mview=Rview Tview：
    • 首先计算平移矩阵 Tview
    • 旋转到规范轴不好求，可以求它的逆矩阵，也就是从规范到相机原来的轴
    • 旋转矩阵的逆矩阵就是它的转置矩阵，就可以求其变换矩阵了

2.2 投影变换

正交投影没有近大远小的现象

  • 正交投影认为相机无限远
  • 透视投影认为相机是一个点，和画面形成一个锥

2.2.1 正交投影

  • 一种简单的理解方式： - 此前说的让相机放在原点，look at -Z, up at Y - 这种方法将 z 轴去掉得到的就会自然是一个二维的平面图 - 进行移动和缩放让所有的结果都映射的[-1，1]^2 的一个小正方形里

    [!question] 如何区分物体的前后？

  • 正式的操作：把空间中的任意长方体（六个数就可以表示，左右，上下，前后坐标[l,r]×[b,t]×[f,n]）映射到一个正则、规范、标准的正方体[-1,1]^3 的正方体中。
    • 只需要把中心平移到原点，然后缩放
    • 一个问题：表示 z 轴区间中远要比近小，这是因为我们是沿着-z 的方向（看向-z，眼眼睛在靠正向一边）去看的

2.2.2 透视投影

  • 最广泛的投影，平行线不再平行
  • 如何进行透视投影：我们想象把视锥的锥形挤压到一个长方体里面
    1. n 平面所有坐标不变
    2. n，f 平面的 z 坐标不变
    3. f 平面最中心的点坐标不变
  • 一种计算技巧：对于齐次坐标的表示，一个点(x,y,z,1),同时乘一个不为 0 的系数(kx,ky,kz,k)依然表示空间中的同一个点，既然如此，同时乘 z 也是一样(xz,yz,z^2,z)也表示同一个点
  • 类似待定系数法求解变换矩阵
  • 最后一步：得到视锥到长方体的矩阵，需要再进行一步正交变换即可得到最后的结果

    [!question] 中间的点挤压后会被推向近的平面还是远的平面
